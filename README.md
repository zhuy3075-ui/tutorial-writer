# tutorial-writer 教程生成 Skill

`tutorial-writer` 是一个面向线下 AI 编程培训的 Codex Skill，用于把本地知识库素材、课程主题、案例复盘或工具说明，转化为结构完整、适合飞书云文档发布的教程。

它的目标不是简单摘录知识库内容，而是先规划课程范围、联网校准陈旧知识、组织知识逻辑链，再生成带有讲解、过渡、练习、图表和飞书排版的课堂教程。

## 主要功能

### 1. 教程范围规划

在正式写教程前，先判断适合做成：

- 单节教程
- 小合集
- 整套课程

并进一步规划飞书承载方式：

- 单篇飞书云文档
- 多篇小节文档 + 总览文档
- Wiki 知识库结构

### 2. 知识库提取与联网校准

Skill 会把知识库当作素材来源，而不是直接拼贴。

它会：

- 提取核心知识点
- 标记可能陈旧的内容
- 必要时联网搜索官方文档、GitHub README、权威资料
- 对比知识库原说法与最新资料
- 把校准后的内容用于教程

它不会自动整理、改写或优化原知识库仓库。

### 3. 多 Agent 深度分析

正式写正文前，要求先进行多 Agent 分析：

- 内容主线 Agent：检查知识点逻辑、过渡、解释深度和资料缺口
- 视觉教学 Agent：判断哪里适合流程图、思维导图、对比图、层级图等
- 资料校准 Agent：在涉及具体工具、API、平台时联网确认版本和限制

### 4. 反拼凑教程写作

教程必须像一节课，而不是资料摘要。

Skill 强制要求：

- 先搭建知识逻辑链
- 每节解释“为什么现在讲它”
- 核心知识点覆盖场景、问题、原理、操作、判断
- 相邻章节不能随意调换
- 每个小节有过渡、练习和验收标准

### 5. 飞书云文档输出

默认输出形态是飞书云文档 `docx`，不是普通 `.md` 文件。

推荐使用：

```bash
lark-cli docs +create --api-version v2 --as user --content @./tutorial.xml --json
```

只有用户明确要求上传原生 Markdown 文件时，才使用：

```bash
lark-cli markdown +create --as user --file ./tutorial.md
```

### 6. 飞书富文本排版

Skill 支持并要求使用适合飞书文档的结构化元素：

- 高亮块
- 表格
- 代码块
- 检查项
- 分栏
- 分隔线
- 画板类图
- 图片插入位

并要求标题层级、序号层级和对比内容正确显示。

### 7. 图表与画板规划

教程中会按实际内容规划：

- 流程图
- 思维导图
- 对比图
- 层级图
- 泳道图
- 决策树
- 因果图

图表必须服务教学逻辑，不能只是装饰。

### 8. img2.0 教程插图

Skill 支持在合适位置规划由 img2.0 生成的教学插图，用于：

- 缓解学习压力
- 演示抽象知识点
- 做阶段汇总

每张图片都会规划：

- 插入位置
- 上下文
- 教学目的
- 生图提示词
- 本地资产路径

并额外生成独立的生图提示词文档，方便复用和迭代。

## 使用方式

在支持 Codex Skill 的环境中，把本仓库中的 `tutorial-writer` 目录安装到 Codex skills 目录。

之后可以这样调用：

```text
使用 $tutorial-writer，把这个知识库主题整理成适合线下课的飞书教程。
```

或：

```text
使用 $tutorial-writer，基于这些资料写一整套课程大纲，并判断应该做成单文档、多文档还是 Wiki。
```

## 适用场景

- AI 编程线下培训教程
- 工具类小白教程
- 飞书云文档课程讲义
- 从知识库素材生成课程脚本
- 从旧知识库内容生成经联网校准的新教程
- 多章节课程或 Wiki 知识库教程规划

## 不适用场景

- 直接整理或重构原知识库仓库
- 只需要简单摘要，不需要课程化表达
- 不需要飞书文档格式的普通 Markdown 写作
- 要求完全不联网且知识库内容明显陈旧的工具/API 教程

## 发布与验证要求

生成飞书文档后，应使用：

```bash
lark-cli docs +fetch --api-version v2 --doc "<doc-url>" --doc-format xml --scope full --json
```

确认：

- 返回链接包含 `/docx/`
- 文档标题正确
- 高亮块、表格、代码块、检查项存在
- 画板或图表插入位存在
- 图片提示词文档已生成
