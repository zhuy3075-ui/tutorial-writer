Codex Skill · Feishu Docx Tutorial Generator

tutorial-writer 教程生成 Skill

把本地知识库、课程主题、案例复盘、工具说明或飞书参考文档，转化为适合发布到飞书云文档的系统化教程。 它会先帮你确认需求、规划结构、检索知识库、校准陈旧内容，再调用飞书 CLI 生成带有高亮块、表格、检查项、画板图、图片和练习任务的正式飞书教程。

适合谁使用

  • 想把本地知识库整理成课程教程的人。
  • 想为线下 AI 编程课生成讲义的人。
  • 想把工具、API 或项目经验写成小白教程的人。
  • 想把教程直接发布成飞书 docx 云文档的人。
  • 想让 Codex 先通过问答帮自己梳理目标、痛点和交付物的人。

核心功能

启动前需求确认

正式生成前先让用户选择教程生成、需求梳理、参考学习或自定义模式，再整理初级需求汇总，确认后才继续。

课程形态规划

判断应该做成单节教程、小合集还是整套课程，并决定单篇文档、多篇文档或 Wiki 知识库结构。

知识库检索与联网校准

从本地知识库提取素材，对可能陈旧的工具/API/平台知识联网校准，只优化教程内容，不修改知识库。

多 Agent 深度分析

正式写正文前，从内容主线、视觉教学、资料校准等角度分析，再组织成有过渡、有解释的教程。

飞书富文本教程

默认生成飞书 docx 云文档，使用 DocxXML 表达高亮块、表格、检查项、分栏、代码块、按钮、画板图和图片。

图表、插图与作业

规划画板图、img2.0 教学插图、互动作业、学习打卡区和真实进度统计入口。

插图原则：img2.0 图片必须回答教程当前位置的具体问题，例如“这一步为什么要先拆任务”“截图、画板、img2.0 分别负责什么”。不能只是“AI 很高级”“资料很多”“云端生产线”这类泛泛概念图。

推荐使用方式：用 Codex 生成

最推荐的流程是：新建 Codex 对话 → 使用计划模式获取需求 → 检索自己的知识库 → 通过对话明确目标 → 调用 $tutorial-writer 生成飞书教程。

第 1 步：新建一个 Codex 对话

建议为每个教程主题新开一个对话，避免上下文混乱。可以先告诉 Codex：

我想做一份飞书教程，请先不要直接生成正文。请先进入计划模式，帮我通过问答确认需求。

第 2 步：使用计划模式获取需求

让 Codex 先问清最终成果、目标学员、当前痛点、课程形态、飞书承载方式、参考风格、是否需要 img2.0 图片、是否需要作业和进度统计。

请使用计划模式帮我梳理一份教程需求。

主题：这里写你的教程主题
知识库路径：这里写你的本地知识库路径
目标读者：这里写学员画像
预期成果：单篇飞书教程 / 小合集 / Wiki 课程 / 公众号式项目实战文

请通过问答确认我的目标、痛点、已有材料、参考风格、交付形式和验收标准。先输出初级需求汇总，不要直接写正文。

第 3 步：让 Codex 检索你的知识库

请检索这个知识库中与本教程主题相关的内容，只提取有用素材，不要修改知识库：

/你的/知识库/路径

请按“可直接用于教程的知识点、可能陈旧需要联网确认的内容、适合做图表的位置、适合做练习的位置”整理。

如果主题涉及快速变化的工具/API，可以补充：

知识库可能有陈旧内容，请联网确认官方最新资料，只用于优化教程，不要改知识库仓库。

第 4 步：通过对话明确目标

目标成果改成小合集。
学员是已经会用 AI 写代码，但不会拆任务的新手。
参考风格偏公众号项目实战，先展示成果，再拆流程。
需要 img2.0 图片，但不要太多。
需要每章有作业和打卡。

确认无误后回复：

继续，按这个需求开始生成。

第 5 步：调用此 Skill 生成飞书教程

请使用 $tutorial-writer，基于刚刚确认的需求、知识库检索结果和联网校准结果，生成正式飞书教程。

要求：
1. 先规划教程形态：单节 / 小合集 / 整套课程。
2. 决定飞书承载方式：单文档 / 多文档 / Wiki。
3. 正文使用自然语言讲解，少用代码堆砌。
4. 使用飞书 DocxXML 富文本块，包括高亮块、表格、检查项、画板图。
5. 需要时生成 img2.0 图片并插入对应位置。
6. 作业区使用 checkbox / 高亮块 / Base 或表单入口。
7. 使用 lark-cli 创建飞书 docx 文档，并返回 /docx/ 链接。
Skill 启动后会再次弹出模式选择或编号选项，这是正常行为。选择“教程生成模式”即可。

第 6 步：确认飞书文档

请用 lark-cli docs +fetch --api-version v2 验证刚生成的飞书文档，确认标题、高亮块、表格、检查项、画板图、图片是否存在。

一条完整示例

我想生成一份飞书教程，请优先使用计划模式帮我获取需求，不要直接写正文。

教程主题：Vibe Coding 小白入门
知识库路径：/Users/zy/Desktop/知识库/本地（Ai库）
目标学员：用过 AI 写代码，但不会拆任务、不会判断输出质量的新手
预期成果：一份适合线下课使用的飞书 docx 教程

请先通过问答整理初级需求汇总。需求确认后，再调用 $tutorial-writer 检索知识库、联网校准，并使用 lark-cli 生成正式飞书教程。

安装方式

把仓库中的 tutorial-writer 目录放到 Codex skills 目录：

mkdir -p ~/.codex/skills
cp -R tutorial-writer ~/.codex/skills/

安装后，在新对话中用 $tutorial-writer 触发。

飞书 CLI 要求

Skill 默认要求生成飞书 docx 云文档。需要本机可用：

lark-cli docs --api-version v2 --help

常用创建命令

lark-cli docs +create --api-version v2 --content @tutorial.xml --json

常用验证命令

lark-cli docs +fetch --api-version v2 --doc "<飞书文档链接>" --doc-format xml --scope full --json

只有用户明确要求上传原生 Markdown 文件时，才使用 lark-cli markdown +create。

使用前准备

  • 本地知识库路径。
  • 教程主题。
  • 目标学员画像。
  • 参考文章或飞书教程链接。
  • 是否需要联网校准。
  • 是否需要 img2.0 图片。
  • 是否需要飞书 Base / 表单做真实进度统计。
  • 飞书 CLI 登录和权限。

不适用场景

  • 只想要一句话摘要。
  • 不需要飞书文档格式。
  • 不希望 Codex 进行需求确认。
  • 不允许联网，但主题强依赖最新工具/API。
  • 想自动修改原知识库仓库。

交付验收清单

  • 已先完成模式选择和需求确认。
  • 已说明教程形态和飞书承载方式。
  • 已检索知识库并提取核心知识点。
  • 陈旧知识已联网校准，或说明无需联网的理由。
  • 正文不是资料拼贴，有清晰过渡。
  • 标题层级、序号层级正确。
  • 有高亮块、表格、检查项、画板图等飞书富文本块。
  • 作业和练习可在课堂内验证。
  • 如规划 img2.0 图片，图片已生成、自检并插入文档。
  • 返回的是飞书 /docx/ 链接。
  • 已用 lark-cli docs +fetch --api-version v2 验证文档内容。
