# 教程课本 Skill 功能规格

## 目标能力

`tutorial-writer` 接收知识库素材、教程主题和课堂背景，输出结构完整、语言通俗、带配图位和练习闭环的飞书教程 Markdown 草稿。

该 Skill 的核心不是“写一篇漂亮文章”，而是把教程写成讲师可讲、学员可做、结果可验证的课堂脚本。

## 输入

执行 Skill 时应尽量获得以下输入：

| 输入项 | 必需性 | 说明 |
|---|---|---|
| 教程主题 | 必需 | 本篇教程要教什么，例如“用 Codex 写一个 Skill”。 |
| 知识库素材 | 必需 | 可来自本地文档、会议记录、产品说明、已有教程或讨论卡片。 |
| 目标受众 | 默认已有 | 默认是有 AI 编程基础的线下学员。若用户另行说明，以用户说明为准。 |
| 环境信息 | 操作型教程必需 | 软件名、版本号、操作系统、写作日期。 |
| 前置阅读 | 必需 | 本篇前一篇教程标题或“无”。 |
| 下一步 | 必需 | 本篇后一篇教程标题或学习路径中的下一步。 |
| 来源链接或来源文件 | 有则必需 | 用于生成 `📎 相关文档：[标题](链接)` 或标注本地来源。 |

缺少教程主题或知识库素材时，应先停止并询问。缺少环境、前置阅读、下一步时，可以在草稿中显式标出需要用户补齐的具体字段，但不能用泛泛的占位词。

## 输出

输出是一篇飞书友好的 Markdown 教程草稿，必须包含：

- H1 教程标题。
- 开头卡片：时间、目标、准备、环境。
- 来源引用：置顶或放在对应段落后，链接文字必须是标题，不放裸 URL。
- 正文结构：三种教程骨架之一，不混用、不自创。
- 每个关键知识点的四段式展开。
- 配图位标注：只写 `[图：...]` 描述，不生成实际图片。
- 至少一个“我栽过的坑” Callout。
- 至少一个课堂讨论节点。
- 5 分钟内可完成并验证的练习。
- 如章节确实需要实操，附现成材料、合格标准和提交方式。
- 末尾包含前置阅读和下一步。

## 工作流

1. 从知识库素材提取 3 到 5 个核心知识点。  
   只提取和教程主题直接相关的内容，不扩展未提供的新方向。

2. 选择一种教程骨架。  
   - 教具体操作流程时，选择“操作指南型”。
   - 教抽象概念时，选择“概念讲解型”。
   - 拆真实项目或经验时，选择“案例拆解型”。

3. 对每个知识点做四段式展开。  
   - 为什么需要这个：用学员已有经验切入，1 到 2 句话。
   - 怎么做：拆成可验证步骤。
   - 最容易犯的错：写真实踩坑，不写想象中的泛泛错误。
   - 一个练习：5 分钟内可完成，完成后能立刻判断结果。

4. 按写作规则改写语言。  
   - 零术语起步。
   - 先例子后定义。
   - 一段不超过四行。
   - 全篇用“你”说话。
   - 每 300 字左右放一个“试试看”。
   - 禁用俯视语言和收费伏笔。

5. 标注配图位。  
   只在读者不看图会卡住的地方标图，必须覆盖点击位置、操作前后对比、容易卡住的界面三类场景。

6. 做飞书格式化。  
   使用 Callout、表格、代码块、折叠块四件套。步骤之间、开头和正文之间、正文和结尾之间使用分割线。

7. 输出前自检。  
   对照质量检查清单逐项确认，不合格项应在输出前修正。

## 教程骨架选择规则

| 判断问题 | 选择骨架 | 输出重点 |
|---|---|---|
| 这篇是在教“怎么做 X”吗？ | 操作指南型 | 步骤、截图位、常见问题、最终效果。 |
| 这篇是在解释“什么是 X / 为什么需要 X”吗？ | 概念讲解型 | 例子、定义、关系、试试看、讨论。 |
| 这篇是在拆“我是怎么做 X / 项目为什么成败”吗？ | 案例拆解型 | 背景、问题、失败尝试、解决路径、可迁移方法。 |

如果一个主题同时包含概念和操作，优先按本篇最主要的学习目标选择一种骨架。不要在同一篇中混合三种骨架；内容太多时拆成多篇。

## 质量规则

### 写作规则

- 专业词第一次出现必须配生活化类比。
- 定义必须出现在例子之后。
- 不使用“众所周知”“显然”“大家都会”“你只需要”“很简单”“不过是”。
- 提到个人成绩时必须说明资源条件，并给出低资源替代方案。
- 免费内容不插入“完整版请购买”式伏笔。

### 结构规则

- 前五行内出现时间、目标、准备、环境。
- 前 200 字内让学员动手或看到可操作入口。
- 每篇必须有练习。
- 每篇末尾必须有前置阅读和下一步。
- 每篇至少一个讨论节点。

### 配图规则

- 只用静图，不用 GIF。
- 标注元素只用红框、数字序号、箭头。
- 配图位写成明确句子，例如：`[图：Codex 左侧 Skill 列表，红框圈出 tutorial-writer，数字 1 标出点击顺序]`。

### 飞书格式规则

- Callout 放提示、注意、常见错误、我栽过的坑。
- 表格放对比和映射关系，不超过 4 列。
- 代码块放命令、代码、配置和 prompt，必须标注语言。
- 折叠块放扩展阅读、补充说明、术语详解和版本差异。

## 验收清单

- [ ] 选定一种教程骨架，且结构完整。
- [ ] 提取了 3 到 5 个核心知识点。
- [ ] 每个关键知识点都有四段式展开。
- [ ] 语言符合 `writing-rules.md` 的适用规则。
- [ ] 配图位符合 `screenshot-guide.md`。
- [ ] 使用了飞书排版四件套。
- [ ] 源文档出处置顶或附在对应段落后。
- [ ] 练习可在 5 分钟内完成并验证。
- [ ] 至少包含一个讨论节点。
- [ ] 实操作业如出现，包含任务、现成材料、合格标准、自检和提交方式。
