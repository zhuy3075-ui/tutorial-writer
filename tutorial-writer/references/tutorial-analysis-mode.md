# tutorial-analysis-mode.md — 参考教程学习与模板结构提炼

这份规范用于用户提供优秀教程、飞书链接、PDF、截图或样稿，并明确目标是“学习优质教程的模板结构”“分析教程设计”“沉淀模板”“优化 skill”时。

这个模式不是教程生成模式，也不是一比一复刻模式。它的目标是从参考教程里提炼可复用结构、图文节奏和教学动作。

## 一、触发条件

遇到以下意图时，进入参考教程学习模式：

- 用户说“学习优质教程模板结构”“分析这篇教程的结构”“提炼可复用模板”。
- 用户要求“只根据教程本身判断”“不要迎合我”“区分明确呈现和推断”。
- 用户提供飞书教程、PDF、截图或优秀文章，希望优化 `tutorial-writer`。
- 用户不是要立刻生成飞书 docx，而是要先研究参考教程。

进入本模式后，不执行默认教程生成硬闸门：不要求先选择快速生成 / 标准诊断 / 深度访谈，也不创建飞书文档。

仍然要遵守：

- 不一比一复制原文原图，除非用户另行确认授权并切换到复刻模式。
- 不把参考教程的行业内容当成事实更新，除非做了来源校准。
- 不凭空补教程没有呈现的内容；推断必须明确标注。

## 二、读取顺序

1. 先读取参考材料目录或封面。
2. 再抽取 2-3 个代表性章节。
3. 再读取图片、视频、附件或截图的结构位置。
4. 最后输出结构卡、节奏卡、模板卡和问题卡。

飞书文档优先使用：

```bash
lark-cli docs +fetch --api-version v2 --doc "<链接>" --doc-format xml --scope outline --max-depth 4 --json
lark-cli docs +fetch --api-version v2 --doc "<链接>" --doc-format xml --scope keyword --keyword "项目完整操作思路|避坑|成品展示|总结|FAQ|注意事项" --context-before 1 --context-after 2 --max-depth 2 --json
```

图片处理：

```bash
lark-cli docs +media-download --token "<token>" --output "./sample"
lark-cli docs +media-preview --token "<token>" --output "./sample"
```

如果 `media-download` 返回 403，记录失败并使用 `media-preview`。如果 preview 也失败，只记录 token、章节位置和图片用途，不截图原图，除非用户确认拥有截图或复刻权限。

PDF 处理：

```bash
pdfinfo "<file.pdf>"
pdftoppm -f 1 -l 3 -png -r 120 "<file.pdf>" "./page"
```

如果 PDF 文本层不可抽取，按渲染页分析版式和页面结构，并在报告里说明限制。

## 三、十段分析框架

分析报告必须包含：

1. 教程一句话概括。
2. 教程目标与受众判断。
3. 教程的设计结构分析。
4. 教程的核心内容拆解。
5. 教程的优秀点分析。
6. 教程的问题点分析。
7. 教程背后的设计思路。
8. 可提炼的精华。
9. 如果要优化这篇教程。
10. 最终独立判断。

每段都要区分：

- **明确呈现**：目录、正文、图片、表格、代码块、视频/附件、封面中直接能看到的内容。
- **推断判断**：基于结构、顺序、图文密度、步骤设计做出的分析。

禁止：

- 只做普通总结。
- 猜测用户想要的结论。
- 把自己的喜好写成教程事实。
- 因为用户说“优秀教程”就只夸不挑问题。

## 四、模板结构提炼输出

分析后必须输出模板卡。每张模板卡包含：

```text
模板名称：
适合主题：
不适合主题：
章节骨架：
开头方式：
图文节奏：
步骤组织：
避坑写法：
案例嵌入：
总结收束：
可复用块：
迁移注意事项：
```

至少提炼：

- 项目实战入门模板。
- 技能训练流程模板。
- 运营课程闭环模板。
- 系统指南工程化模板。

如果参考材料不足以支持某个模板，必须写“证据不足”，不要强行生成。

## 六、沉淀到 skill 的规则

只有当用户明确要求“优化 skill”“长期加入模板”“沉淀为 SOP”时，才修改 `tutorial-writer`。

沉淀时优先：

- 新增 reference 文件，而不是把长分析塞进 `SKILL.md`。
- 在 `SKILL.md` 只加入触发、路由和必须加载的 reference。
- 保持生成教程、参考复刻、修改已有文档三个流程互不混淆。

修改完成后验证：

- `参考教程学习` 不触发飞书 docx 创建。
- `教程复刻` 仍要求总结借鉴 / 混合复刻 / 一比一还原选择。
- `生成新教程` 仍走需求诊断和大纲审查。
- `修改已有飞书文档` 仍默认更新原文档。
