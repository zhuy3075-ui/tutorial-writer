# course-planning.md — 教程范围、章节部署与飞书承载规划

这份规范解决一个前置问题：收到主题后，不要立刻写正文。先通过 `intake-and-mode.md` 完成模式选择，再通过 `needs-diagnosis.md` 完成项目级需求诊断结果；用户确认后，再判断这次应该做“一小节”“一个合集”还是“一整套课程”，最后决定飞书里怎么组织。

当范围判断或飞书承载方式需要用户选择时，必须按 `choice-interaction.md` 弹出可点击选项；不可用时才编号降级，并停止等待用户回复。

## 一、不要优化知识库仓库

知识库是素材来源，不是本 skill 的整理对象。

允许做：

- 读取知识库材料，提取观点、案例、步骤、术语。
- 标记哪些知识可能陈旧、冲突或缺少证据。
- 联网校准事实、版本、工具入口、API、最佳实践和限制。
- 在教程里说明“知识库材料与最新资料的差异”。

禁止做：

- 为了写教程而重构知识库目录。
- 删除、改写、合并知识库原文。
- 把联网资料回填到知识库，除非用户明确要求。
- 把“知识库清理”混进“教程写作”任务。

## 二、范围判断：单节 / 合集 / 整套课

正式写教程前，先做范围判断。

范围判断必须基于已确认的“项目级需求诊断结果”，不能只根据主题名猜。至少参考：

- 目标成果：用户到底想要单篇教程、合集、课程还是参考模式。
- 目标受众：学员基础和使用场景。
- 当前痛点：是只卡一个操作，还是缺完整能力路径。
- 已有材料：知识库、飞书链接、截图、案例是否足够。
- 验收标准：最终要能做出什么、看见什么、交付什么。
- 审美表达：更像操作指南、公众号项目实战文、线下讲义还是 Wiki。
- 发布风险：权限、版本、陈旧知识、素材缺失、飞书能力限制。

| 形态 | 适用场景 | 产出 |
|------|----------|------|
| 单节教程 | 目标明确、30 分钟内能讲完、只解决一个任务 | 1 篇飞书云文档 |
| 小合集 | 主题有 2-5 个连续能力点，但还不是完整课程 | 1 篇总览文档 + 2-5 篇小节文档 |
| 整套课程 | 需要从入门到实战，包含多个模块、作业、复盘 | Wiki 知识库或课程文件夹 |

判断问题：

```text
这次主题是解决一个具体任务，还是培养一组能力？
学员学完后只需要会做一件事，还是要完成一个项目？
内容是否超过 5 个核心知识点？
是否需要分课时、作业、复盘和阶段验收？
是否需要后续持续维护？
```

如果答案偏向“一个具体任务”，做单节。  
如果答案偏向“几个连续能力点”，做小合集。  
如果答案偏向“完整学习路径”，做整套课程。

如果判断不唯一，必须给用户可点击选择：

```text
1. 单节教程（推荐）：先解决一个明确任务，交付最快，适合首次验证。
2. 小合集：把 2-5 个连续能力点拆成总览和小节，结构更系统。
3. 整套课程：适合长期维护和训练营，但需要更多素材、作业和验收设计。
```

## 三、先出课程框架，再写小节

无论最终做单节还是整套课，都先输出框架。

## 四、飞书承载方式决策

不要默认所有内容都塞进同一个文档。

| 内容规模 | 推荐承载 | 原因 |
|----------|----------|------|
| 单节教程 | 单篇飞书云文档 | 阅读连续，维护简单 |
| 2-5 节小合集 | 总览文档 + 多篇小节文档 | 总览负责导航，小节负责教学 |
| 完整课程 | Wiki 知识库 / 课程文件夹 | 层级清楚，适合长期维护 |
| 临时课堂讲义 | 单篇长文档 | 方便一次性分享 |
| 可持续迭代课程 | Wiki | 方便按模块更新 |

如果需要用户在单文档、多文档和 Wiki 之间做取舍，必须提供可点击选项，并写清维护成本和发布风险。

### 单文档适合

- 30 分钟以内能讲完。
- 不超过 5 个核心知识点。
- 不需要跨章节跳转。
- 作业只有 1 个。

### 多文档适合

- 每节都有独立练习或作业。
- 内容超过 5 个核心知识点。
- 有入门、进阶、实战、复盘等阶段。
- 后续可能继续扩展。

### Wiki 知识库适合

- 是一整套课程或训练营。
- 需要目录、模块、小节、作业、素材区。
- 需要多人协作维护。
- 需要长期更新版本。

## 五、章节部署规范

“部署”不是技术部署，而是课程内容如何安放。

注意：这里的“章节部署”只表示课程内容的组织方式。如果教程主题涉及项目上线、发布、交付给 Codex / WorkBuddy / Claude Code 等技术部署流程，必须切换到 `deployment-handoff.md`，不要沿用本节规则。

### 总览文档必须包含

- 课程目标
- 学习路径图
- 章节目录
- 每节产出
- 前置要求
- 作业提交方式
- 更新记录或版本说明

### 每个小节文档必须包含

- 本节目标
- 本节在整套课里的位置
- 上一节产出了什么
- 本节解决什么新问题
- 本节最终产出
- 练习与验收标准
- 下一节为什么要学

### Wiki 结构建议

```text
课程首页
├── 00 课程导览
├── 01 入门与心智模型
│   ├── 01-01 第一节
│   └── 01-02 第二节
├── 02 核心操作
│   ├── 02-01 第一节
│   └── 02-02 第二节
├── 03 实战项目
├── 04 作业与验收
└── 99 素材与扩展阅读
```

如果当前环境没有 Wiki 权限或用户没有指定 Wiki 空间，先生成“Wiki 结构规划”和飞书云文档，不擅自创建复杂 Wiki。

## 六、联网校准流程

知识库材料可能陈旧时，按这个流程处理：

```text
1. 从知识库提取候选知识点
2. 标记可能过期的点：工具版本、命令、API、价格、入口、限制、最佳实践
3. 联网查官方/权威资料
4. 对比：保留 / 修正 / 删除 / 标注为历史经验
5. 把校准结果写进教程，不改知识库原文
```

校准输出格式：

```text
知识点：______
知识库说法：______
联网确认：______
采用结论：______
放入教程的位置：______
来源：______
```

## 七、规划验收

写正文前必须确认：

- [ ] 已完成模式选择、项目类型判断和项目级需求诊断结果确认。
- [ ] 已判断单节 / 小合集 / 整套课。
- [ ] 已说明为什么选择这种范围。
- [ ] 已决定单文档 / 多文档 / Wiki。
- [ ] 已规划章节与小节的层级。
- [ ] 每个小节都有“承接上一节”和“引出下一节”。
- [ ] 已标记需要联网校准的知识点。
- [ ] 联网校准只用于教程，不修改知识库仓库。
