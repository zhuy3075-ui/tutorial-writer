# choice-interaction.md — 全流程可点击选项规则

这份规范约束所有需要用户做选择、确认、授权、审查或风险接受的节点。目标是让用户像在计划模式里点选方案，而不是反复手动回复序号。

## 一、硬规则

- 只要问题能被整理成有限选项，必须先尝试使用真实可点击选项：弹窗、选项卡、计划模式选项、`request_user_input` 或当前运行环境提供的等价交互。
- 不允许在可点击选项可用时只发纯文本编号列表。
- 每个选项必须有 `label` 和 `description`；`description` 要写清楚适用场景、收益和代价。
- 推荐项必须放在第一位，标签带“推荐”或在描述里写明推荐理由。
- 发送选项后必须停止推进，等待用户点击或选择；不得继续读取知识库、联网、写正文、生成图片、创建或更新飞书文档。
- 如果当前环境没有真实可点击选项能力，才允许降级为编号选项；降级时仍要写清每个选项的描述，并停止等待用户回复。
- 不能用普通文字假装已经弹窗。是否用了真实可点击选项，要在后续自检里可追溯。

## 二、必须使用选项的节点

以下节点必须按本规范处理：

- 启动模式选择：快速生成、标准诊断、深度访谈、参考学习、修改已有飞书文档、自定义模式。
- 项目类型判断：单篇教程、小合集、Wiki 课程、项目实战文、工具操作教程、部署交付教程、线下课讲义、参考文档复刻。
- 需求诊断后的确认：确认继续、返回修改、进入深度访谈、缩小范围。
- 教程形态和承载方式：单文档、多文档、Wiki、更新原文档、另存副本。
- 参考飞书处理：总结借鉴、混合复刻、一比一还原。
- 权限确认：是否拥有复制、复刻、课堂使用、截图回填、保留水印或隐私元素的权限。
- 大纲审查：通过生成、调整章节、增删章节、改风格、返回需求诊断。
- 图文资产选择：是否生成 img2.0、生成几张、使用截图/画板/表格/新图替代。
- 风险接受：跳过需求诊断、跳过大纲审查、无法联网、无法读取参考文档、飞书能力不支持某块。
- Quality Loop 超限后的处理：继续修、接受当前版本、缩小范围、等待用户补材料。

## 三、选项写法

选项内容必须短而具体，不要把问题丢回给用户。

```text
label: 标准诊断（推荐）
description: 先做项目级需求诊断和大纲审查，能兼顾质量和速度，适合正式飞书教程。
```

二选一确认也要做成选项：

```text
label: 确认继续（推荐）
description: 需求已经足够清楚，下一步生成全流程规范大纲。

label: 返回修改
description: 先调整目标、学员水平、审美或交付方式，再重新确认需求。
```

高风险选择必须把代价写明：

```text
label: 安全方案（推荐）
description: 只总结借鉴结构和排版，不复制原文原图，版权和权限风险最低。

label: 一比一还原
description: 尽量复制原文、原图和格式；只有你确认拥有复制或课堂使用权限时才能继续。
```

## 四、降级规则

当真实可点击选项不可用时，使用编号降级：

```text
当前环境没有可点击选项能力，我先用编号选项降级处理。请选择一个后我再继续：

1. 标准诊断（推荐）：先诊断再出大纲，质量最稳。
2. 快速生成：只补关键缺口，更快但个性化较弱。
3. 深度访谈：多轮理解真实需求，更贴合但耗时更长。
```

降级后必须等待用户回复，不得自行选择。最终报告或阶段记录里要说明“本轮因环境限制使用了编号降级”。

## 五、开放问题边界

需求访谈可以问开放问题，但如果问题本质是让用户在几种路径之间选，就必须转成可点击选项。

- 可以开放问：你希望学员看完后能独立完成什么？
- 必须选项问：这次更像单篇教程、小合集还是 Wiki 课程？
- 可以开放问：你最担心哪一步讲不清楚？
- 必须选项问：大纲通过、调整章节、改风格还是返回需求诊断？

如果用户选择“其他”或用自由文本补充，先把它整理成新的已选方案；只有含义仍然模糊时，才继续追问。

## 六、自检

进入正文生成前确认：

- [ ] 所有选择、确认、授权、审查和风险接受节点都优先尝试了真实可点击选项。
- [ ] 每个选项都有 `label` 和 `description`。
- [ ] 推荐项放在第一位且写明理由。
- [ ] 发送选项后已停止等待，没有继续执行下游动作。
- [ ] 如使用编号降级，已说明环境限制，并等待用户回复。
