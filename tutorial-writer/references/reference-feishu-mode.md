# reference-feishu-mode.md — 参考飞书教程复刻与借鉴模式

这份规范处理用户提供“别人/其他团队/优秀教程”的飞书链接时，应该如何选择复刻深度、处理版权权限、提取图片并把内容放回目标教程。

## 一、先让用户选择复刻深度

用户提供参考飞书教程链接后，不能默认一比一还原，也不能默认只总结。必须先给用户选择。

必须按 `choice-interaction.md` 先用可点击选项；如果环境不支持，再用带描述的编号列表降级，并停止等待用户回复。

```text
我先确认参考方式：

1. 总结借鉴（推荐）：只学习结构、排版、图文节奏和教学设计，不复制原文原图；适合参考别人的公开教程。
2. 混合复刻：保留类似版式和图片位置，但正文重写成你的主题；适合想要同款阅读体验。
3. 一比一还原：尽量复制原文、原图、顺序、格式和图片位置；只适合你拥有复制/复刻/课堂使用权限的文档。
```

如果用户选择“一比一还原”，继续用可点击选项确认权限：

```text
1. 我有权限，继续：你确认拥有复制、复刻或课堂使用这份飞书文档内容和图片的权限，可以做一比一还原。
2. 改为总结借鉴（推荐）：只学习结构、排版和图文节奏，不复制原文原图，权限风险最低。
3. 改为混合复刻：保留类似版式和图片位置，但正文与图片改成本次教程自己的内容。
```

## 二、三种模式的执行边界

| 模式 | 内容处理 | 图片处理 | 适用场景 |
| --- | --- | --- | --- |
| 总结借鉴 | 提炼结构、节奏、标题层级、飞书块用法；正文重新写 | 不复制原图，只描述图片作用或生成新图 | 公开教程、无授权参考、学习风格 |
| 混合复刻 | 复用类似排版和章节节奏；正文改写为本次主题 | 保留同类图片位置，用用户素材、截图或新生成图片替代 | 想要同款效果但不照抄 |
| 一比一还原 | 在授权前提下复制内容、顺序、格式和图片位置 | 优先下载原图；下载失败且可见时截图回填 | 用户自有文档、内部授权文档、课堂复刻 |

没有用户授权时，禁止一比一复制大段原文、原图、截图或完整排版细节。可以总结、概括、抽取结构和可复用规则。

## 三、读取参考文档

读取前遵守 `lark-doc` skill 和 v2 API 规则。

先取目录和结构：

```bash
lark-cli docs +fetch --api-version v2 --doc "<参考飞书链接>" --doc-format xml --scope outline --max-depth 4 --json
```

需要复刻或分析图片位置时，取完整结构和 block id：

```bash
lark-cli docs +fetch --api-version v2 --doc "<参考飞书链接>" --doc-format xml --detail full --json
```

记录以下映射：

```text
参考文档资产映射

章节：
原 block id：
目标位置：
原块类型：标题 / 段落 / callout / table / image / whiteboard / grid / checkbox / pre
图片 token 或 url：
图片获取方式：url / media-download / media-preview / screenshot / placeholder
处理模式：总结借鉴 / 混合复刻 / 一比一还原
权限状态：已确认 / 未确认 / 不适用
```

## 四、图片处理规则

图片处理顺序：

1. XML 中 `<img>` 有 `url`：优先用 URL 下载。
2. 有图片 token：先用 `lark-cli docs +media-download` 下载。
3. `media-download` 失败或 403：尝试 `lark-cli docs +media-preview`。
4. 下载/预览都不行，但用户确认有权限、且图片在浏览器或飞书客户端中可见：用浏览器控制、Playwright、Chrome、computer use 或系统截图把该图片截下来。
5. 无权限或不可见：不截取原图；用图片占位、图注说明或重新生成与教程主题相关的新图。

截图回填要求：

- 截图必须只截取目标图片或必要区域，不截无关聊天、用户头像、评论、侧边栏、通知和隐私信息。
- 截图文件要按章节命名，保存到本次教程资产目录。
- 截图插入目标教程时，放在原参考文档对应的相对位置：同一章、同一段落之后或同一说明之前。
- 截图前后要保留原图的教学作用：图前说明为什么看，图后说明怎么看。
- 如果图片中有版权水印、个人信息、二维码、手机号、邮箱、头像墙等敏感内容，先提醒用户确认是否保留；不能擅自清除水印。

## 五、内容和格式回填规则

### 总结借鉴

输出“参考文档模式卡”，只保留：

- 开头方式。
- 章节顺序。
- 标题层级。
- 图文节奏。
- 常用飞书块。
- 图片/画板/表格使用方式。
- 过渡方式。
- 练习与验收方式。
- 可复用规则。

正文必须围绕本次主题重写，不贴原文。

### 混合复刻

允许保留：

- 相似章节节奏。
- 相似视觉块类型。
- 相似图片位置。
- 相似作业与总结方式。

必须改写：

- 正文内容。
- 案例。
- 学员任务。
- 图注。
- 关键判断标准。

### 一比一还原

只有用户确认权限后才执行。要求：

- 尽量保持标题层级、段落顺序、表格结构、callout 类型、代码块、图片位置一致。
- 原图可下载则下载后插入；不可下载但可见且授权则截图插入。
- 若某个飞书块无法完全复刻，使用最接近的 DocxXML 块，并在最终报告说明差异。
- 复刻后用 `docs +fetch --api-version v2` 验证目标文档结构和图片数量。

## 六、失败处理

如果参考文档无权限：

- 说明无法读取。
- 让用户提供导出文件、截图或授权后的链接。
- 不凭空猜测内容。

如果图片无法下载也无法截图：

- 在目标文档相同位置放图片占位。
- 图注写清楚：`[图片占位：原参考图无法读取/下载，需要用户补图]`。
- 不用无关 img2.0 图片冒充原图。

如果用户要求复刻但权限未确认：

- 停止一比一复刻。
- 用可点击选项给出两个替代方案：总结借鉴、混合复刻。

## 七、验收清单

- [ ] 已用可点击选项让用户选择总结借鉴 / 混合复刻 / 一比一还原；不可用时才编号降级。
- [ ] 一比一还原前已确认用户拥有复制、复刻或课堂使用权限。
- [ ] 已用 `docs +fetch --api-version v2` 读取参考文档结构。
- [ ] 已建立图片/块位置映射。
- [ ] 图片优先按 URL/token 下载；下载失败再 preview；仍失败且授权可见时才截图。
- [ ] 截图不包含无关隐私信息。
- [ ] 图片插入目标教程的同一相对位置。
- [ ] 未授权时只总结借鉴结构，不复制大段原文或原图。
