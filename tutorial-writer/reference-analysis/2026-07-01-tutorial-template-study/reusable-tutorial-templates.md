# 可复用教程模板库

## 使用原则

- 模板只提供结构，不替代判断。
- 套模板前先判断教程目标、学员水平、素材类型和交付场景。
- 每个模板都要补“明确呈现的证据”和“基于内容的推断”，避免把参考文档风格误当成普适规律。

## 模板一：项目实战入门模板

适合主题：

- AI 编程小项目。
- 工具驱动的网页、H5、自动化、小应用。
- 需要展示成品、拆流程、带学员跟做的课程。

不适合主题：

- 纯概念科普。
- 没有明确成品的观点文。
- 需要严谨 API 说明的技术参考手册。

章节骨架：

```text
1. 项目介绍
2. 最终效果 / 成品展示
3. 项目完整操作思路
4. 前置准备
5. 产品需求挖掘 / 和 AI 讨论需求
6. 项目实操步骤
7. 调试与改 BUG
8. 成品验收
9. 新手避坑
10. 总结与下一步
11. FAQ
```

图文节奏：

- 开头 10% 内展示成品或完整路径图。
- 每个步骤配真实截图或操作结果。
- 大段 prompt 前后都要有解释：为什么这样问、怎么判断 AI 回答对不对。
- 成品展示不能只放图，要写“这个成品证明了哪些能力”。

可复用块：

- 项目完整操作思路卡。
- 前置准备清单。
- 和 AI 对齐需求的提示词。
- 调试截图 + 向 AI 反馈问题模板。
- 成品验收表。
- 新手避坑清单。

适合迁移的教学动作：

- 先做一个最小项目，再做进阶项目。
- 先让 AI 复述需求和提问，再让 AI 写代码。
- 把报错截图或控制台信息交给 AI，而不是让学员死磕。

## 模板二：技能训练流程模板

适合主题：

- AI 视频、AI 绘图、AI 短剧、设计、剪辑等创作技能。
- 需要先理解基础概念，再逐步产出作品。

不适合主题：

- 只需要账号注册或单一按钮操作的短教程。
- 商业模式/政策解读占主导的课程。

章节骨架：

```text
1. 基础理论知识
2. 核心流程总览
3. 工具选择与素材准备
4. 案例演示
5. 分步骤产出
6. 质量判断标准
7. 后期处理
8. 作品展示
9. 常见问题
10. 练习与复盘
```

图文节奏：

- 概念之后立刻给流程图或示意图。
- 每个生成环节都要展示输入、输出和筛选标准。
- 对审美类内容，必须写“为什么这个结果更好”。

可复用块：

- 术语零基础解释。
- 流程类比图。
- 提示词结构拆解。
- 好结果/坏结果对比。
- 作品后期处理清单。

适合迁移的教学动作：

- 把抽象技能拆成连续产物：剧本、分镜、图片、视频、剪辑。
- 先讲“判断标准”，再让读者生成作品。
- 允许结果不完全一致，但要求核心结构一致。

## 模板三：运营课程闭环模板

适合主题：

- YouTube、抖音、小红书、公众号等内容运营。
- 账号、平台机制、变现、数据分析和政策风险都重要的课程。

不适合主题：

- 只讲单个工具操作的教程。
- 不涉及平台规则和商业闭环的创作课。

章节骨架：

```text
1. 项目介绍
2. 平台机制
3. 变现方式
4. 开始前注意事项
5. 账号注册 / 频道创建
6. 找赛道和对标
7. 内容制作方法
8. 数据分析与复盘
9. 政策风险与申诉
10. 案例分享
11. 工具与自动化提效
```

图文节奏：

- 先建立商业判断，再进入操作。
- 机制类内容用表格或流程图。
- 风险类内容用“情况 -> 原因 -> 处理动作”。
- 案例必须说明为什么能复用、哪里不能照搬。

可复用块：

- 平台机制速读卡。
- 变现路径对比表。
- 账号准备清单。
- 对标库模板。
- 数据指标解释表。
- 政策风险 FAQ。

适合迁移的教学动作：

- 先讲平台如何分发，再讲内容怎么做。
- 先讲收益路径，再讲账号和素材准备。
- 用真实案例训练判断，不只给操作步骤。

## 模板四：系统指南工程化模板

适合主题：

- Agent Skills、MCP、Prompt 工程化、工作流沉淀、工具体系搭建。
- 读者已有基础操作经验，需要升级为可复用方法。

不适合主题：

- 完全零基础工具入门。
- 只需要交付单次结果的短教程。

章节骨架：

```text
1. 封面定位：这份指南解决什么问题
2. 前置要求：读者已经会什么
3. 学习路线：从概念到实战的路径
4. 核心概念：先解释为什么需要它
5. 创建/安装流程
6. 典型案例
7. 高级用法
8. 质量标准
9. 复用与维护
10. 附录 / 资源
```

图文节奏：

- 首屏先给定位、前置要求、篇幅和实战案例数量。
- 概念解释要服务后面的实操。
- 案例要展示“如何从一次经验沉淀成长期能力”。

可复用块：

- 读者定位卡。
- 前置能力清单。
- 从概念到实战路径图。
- 典型案例索引。
- 质量检查表。

适合迁移的教学动作：

- 先说明“不适合谁”，降低误用。
- 把能力边界写在开头。
- 用案例数量和实战范围建立信任。

## 模板选择决策表

| 用户目标 | 推荐模板 | 不要选的模板 |
| --- | --- | --- |
| 带学员做出一个 AI 小工具 | 项目实战入门模板 | 运营课程闭环模板 |
| 教 AI 视频或图片创作流程 | 技能训练流程模板 | 系统指南工程化模板 |
| 教账号、平台和变现闭环 | 运营课程闭环模板 | 项目实战入门模板 |
| 教如何沉淀 Skill / 工作流 | 系统指南工程化模板 | 技能训练流程模板 |

## 写作迁移清单

- [ ] 开头是否 5 行内说明适合谁、学完能做什么、需要准备什么。
- [ ] 是否有“完整操作思路”或等价路径图。
- [ ] 是否把前置条件单独成块，而不是埋在步骤里。
- [ ] 每个步骤是否有输入、动作、可见结果、失败处理。
- [ ] 图片是否服务“看哪里 / 看什么 / 怎么判断”。
- [ ] 避坑是否来自真实失败路径。
- [ ] 总结是否回收到能力变化，而不是泛泛鼓励。
