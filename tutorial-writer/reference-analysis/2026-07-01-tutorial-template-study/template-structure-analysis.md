## 一、教程一句话概括

明确呈现：

- AI 短剧教程主要教新手从影视基础、剧本分镜、图片/视频生成到后期剪辑，完成 AI 影像作品制作。
- YouTube AI 视频教程主要教新手理解平台机制、账号准备、内容制作、变现和数据/风险处理。
- Claude Code 教程主要教新手认识 Claude Code、配置环境，并通过多个小项目学习 AI 编程应用。
- Agent Skills PDF 主要讲如何把工作流程沉淀成 Agent Skills，让 AI 记住并复用你的工作方式。

推断判断：

- 四份材料共同解决的不是“知道某个概念”，而是“把一个复杂领域拆成可跟做、可复用、可迁移的学习路径”。

## 二、教程目标与受众判断

明确呈现：

- 三份飞书教程都直接面向新手，标题中多次出现“新手如何入门”“基础理论”“项目实操步骤”“常见问题”。
- Claude Code 教程把 Git、VS Code、终端、Node.js 等前置概念单独拆出来，说明目标受众并非传统开发者。
- YouTube 教程先讲平台机制和变现，再讲账号、对标、制作、数据和政策，面向想跑通商业闭环的人。
- PDF 封面标注“从概念到实战、从安装到创作”，并给出前置要求：使用过 Claude Code / Cursor / Codex 等 AI 编程工具。

推断判断：

- 飞书教程更适合零基础到初阶用户，核心是降低开始门槛。
- PDF 更适合已有 AI 工具经验的进阶用户，核心是把经验工程化。
- 优质教程的共同点是先给读者定位：你是谁、你现在缺什么、学完能得到什么。

## 三、教程的设计结构分析

明确呈现：

- Claude Code 教程结构多次重复“项目情况 -> 完整操作思路 -> 操作重点 -> 新手避坑 -> 开船前必备 -> 章节实操 -> 成品展示 -> 总结/FAQ”。
- AI 短剧教程从“基础理论知识”进入，再到“剧本和分镜”“人物和环境图片”“图生图”“生视频”“后期剪辑”。
- YouTube 教程先讲“项目介绍 / 完整操作思路”，再讲平台机制、变现方式、账号注册、找赛道、视频制作、数据分析、政策风险。
- PDF 首屏先给定位、前置要求、篇幅和实战案例数量，属于“先承诺学习成果和适用人群”的系统课程入口。

推断判断：

- 三份飞书教程不是单篇文章，而是课程资料库：它们靠大目录承载多条学习路径。
- 最值得复用的是“先总览，后分支”的结构：先把完整路径摆出来，再拆每一段。
- 最大结构问题是标题层级普遍被 `<h1>` 扁平化承载，机器读取时层级不够干净，后续 skill 应主动重建 H1/H2/H3/H4。

## 四、教程的核心内容拆解

必须掌握的核心步骤：

- 项目型教程：项目介绍 -> 操作思路 -> 前置准备 -> 步骤实操 -> 产出展示 -> 避坑/FAQ。
- 技能型教程：基础概念 -> 关键流程 -> 工具选择 -> 案例演示 -> 作品产出 -> 后期处理。
- 运营型教程：平台机制 -> 变现路径 -> 账号准备 -> 内容制作 -> 数据复盘 -> 政策风险。
- 系统指南型教程：定位 -> 前置条件 -> 概念模型 -> 安装/创建 -> 实战案例 -> 复用规范。

辅助理解的背景信息：

- 平台机制、行业趋势、变现方式、工具生态、前置概念解释。
- 视频学习链接、截图位置说明、外部资料链接和参考案例。

作者经验或隐含技巧：

- 先用“完整操作思路”让读者看到全局，再进入细节。
- 大量使用真实截图/成品图降低不确定性。
- 把新手最容易失败的地方提前写成避坑建议。
- 在复杂工具教程里反复强调“截图问 AI”“不要死磕”“先做简单案例”。

可以沉淀的通用方法：

- “路径图先行”：每篇教程前 10% 必须出现完整路径。
- “前置门槛拆解”：新手可能缺的工具、账号、概念要单独成章。
- “每步有可见结果”：步骤后必须写明看到什么才算对。
- “避坑前置”：不要等 FAQ 才讲风险，关键步骤前就要提示。

## 五、教程的优秀点分析

明确呈现：

- Claude Code 教程图片密度极高：约 18 万字符中有 638 个 `<img>`，大量步骤都有截图支撑。
- AI 短剧教程用“五大环节”“剧本 -> 分镜 -> 图片 -> 视频 -> 剪辑”的流程降低复杂度。
- YouTube 教程把赚钱路径、账号安全、内容制作、数据分析、政策风险都放进同一学习路径。
- PDF 用封面定位、前置要求和案例数量先建立读者预期。

推断判断：

- 优秀点不是“内容多”，而是能把新手的不确定性拆成连续台阶。
- 这些教程的强项是实操导向：读者知道下一步要点哪里、准备什么、看到什么。
- 最值得学习的是“课程式长文”的组织能力，而不是某一段具体文字。

## 六、教程的问题点分析

明确呈现：

- 三份飞书教程大量标题都以 `<h1>` 呈现，结构层级在机器读取时不稳定。
- 部分章节夹杂联系方式、素材链接、视频链接、附件和内容正文，信息类型没有完全分层。
- YouTube 教程内容跨度极大，从平台、变现、账号到政策，容易让新手迷失主线。
- Claude Code 教程图片很多，但部分段落依赖“如下方截图”“视频位置如下方截图”，如果图片失效，文字独立性会下降。
- PDF 文本层不可抽取，本轮只能按渲染页做视觉结构分析。

推断判断：

- 这些教程强在“全”，弱在“导航”和“结构清洁度”。
- 对新手最不友好的地方是大体量材料容易变成资料库，而不是单次可完成任务。
- 后续 `tutorial-writer` 生成时应吸收其结构，不应照搬其信息密度。

## 七、教程背后的设计思路

文章中直接体现：

- 先提出项目/课程入口，再给完整操作思路。
- 先讲基础概念，再让读者进入实操。
- 通过真实案例和截图带出操作方法。
- 通过避坑、注意事项、FAQ 降低失败成本。
- 通过视频链接、图文截图、提示词和附件形成多材料学习包。

基于内容的推断：

- 设计者更像在写“航海手册/训练营讲义”，不是普通博客文章。
- 目标不是一次读完，而是让学员跟着阶段完成任务。
- 结构优先服务课堂/社群陪跑，而不是搜索引擎阅读。

## 八、可提炼的精华

最有价值的观点：

- 优质教程先解决“我为什么学、要走哪条路”，再解决“怎么做”。
- 大体量课程必须先给路径图，否则截图和步骤会变成噪音。
- 新手教程要承认不确定性：截图不同、AI 输出不同、版本不同都正常。

最关键的操作方法：

- 每篇教程开头固定输出“完整操作思路”。
- 每个实操段落按“输入材料 -> 操作动作 -> 可见结果 -> 失败处理”写。
- 每个复杂工具教程必须有“前置准备”和“常见错误”。

最值得复用的结构：

- 项目实战模板：项目介绍 -> 完整操作思路 -> 前置准备 -> 实操步骤 -> 成品展示 -> 避坑 -> FAQ。
- 技能训练模板：基础理论 -> 工具/流程 -> 案例演示 -> 作品产出 -> 后期处理。
- 运营课程模板：平台机制 -> 变现方式 -> 账号准备 -> 内容制作 -> 数据复盘 -> 风险应对。
- 系统指南模板：定位 -> 前置要求 -> 概念模型 -> 安装/创建 -> 实战案例 -> 复用规范。

最适合二次内容化的部分：

- “完整操作思路”适合做短视频/图文卡片。
- “避坑指南”适合做小红书/公众号短内容。
- “项目实操流程”适合做直播课和录屏课。
- “FAQ”适合做社群答疑库。

## 九、如果要优化这类教程

标题如何更清楚：

- 从“新手如何入门 X 项目”改成“从 0 跑通 X：准备、实操、成品和避坑”。
- 标题应同时说明对象、路径和结果。

开头如何更吸引人：

- 先展示最终产出或完整路径图，再解释为什么值得学。
- 开头 5 行内说清：适合谁、学完能做什么、需要准备什么。

结构如何调整：

- 把“联系方式/备用链接/视频学习链接”从主线正文中剥离到资料区。
- 把重复 `<h1>` 层级整理成 H1/H2/H3/H4。
- 每个大模块前增加“本模块学完能做什么”和“下一模块为什么出现”。

需要补充案例的位置：

- 运营课程的政策、限流、数据复盘章节应补“错误案例 -> 判断 -> 处理动作”。
- Claude Code 的环境安装部分应补“失败界面 -> 先查什么 -> 怎么向 AI 提问”。
- AI 短剧的分镜/视频生成部分应补“好提示词 vs 坏提示词”的对比。

需要拆细的步骤：

- 所有“复制提示词发送给 AI”都应拆成：给什么上下文、让 AI 先问什么、如何验收第一版。
- 所有“成品展示”都应补：成品是否达到目标、下一轮如何优化。

可以删减的内容：

- 与本章节任务无关的长背景。
- 重复出现的泛泛鼓励语。
- 只有链接但没有学习目的的素材入口。

如何让读者更容易做出结果：

- 每章末尾加“本章最小作业”和“合格标准”。
- 每个截图前写“这张图证明什么”，截图后写“下一步看哪里”。
- 每个长教程增加“30 分钟最短路径”和“完整进阶路径”两条路线。

## 十、最终独立判断

这是不是好教程：

- 是，但不是因为每一段都精致，而是因为它们提供了完整学习路径、真实材料和大量可跟做证据。

最大价值：

- 把复杂领域拆成可执行路径，让新手能从“完全不知道怎么开始”变成“知道下一步做什么”。

最大问题：

- 结构层级和信息分区不够干净，容易从教程变成资料库。

最适合提炼成什么形式：

- 模板库、SOP、课程骨架、图文节奏规则和参考教程分析模式。

只能保留三个精华点：

- 完整操作思路必须前置。
- 每个实操步骤都要有截图/产出/判断标准。
- 避坑和 FAQ 要服务新手的真实失败路径，不做装饰。
